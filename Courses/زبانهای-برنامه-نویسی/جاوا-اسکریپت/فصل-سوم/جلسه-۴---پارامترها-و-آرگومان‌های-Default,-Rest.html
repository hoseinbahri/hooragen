جلسه ۴ پارامترها و آرگومان‌های Default, Rest

جلسه ۴ - پارامترها و آرگومان‌های Default, Rest

بررسی Default Parameters برای تعیین مقادیر پیش‌فرض و Rest Parameters برای دریافت تعداد نامشخصی از آرگومان‌ها و مزایای استفاده از Rest Parameters نسبت به شیء arguments

نوشته شده در: ۰۳ بهمن ۱۴۰۴زمان تقریبی مطالعه: ۴ دقیقه

تحلیل پاسخ‌های تو:

سوال ۱: تابع multiply

  • ✅ کاملاً درست. از سینتکس Arrow Function با Implicit Return به درستی استفاده شده است. بسیار مختصر و تمیز.

سوال ۲: تبدیل تابع square

  • ✅ کاملاً درست. از تمام قوانین کوتاه‌نویسی به خوبی استفاده کردی: حذف پرانتز برای یک پارامتر و حذف آکولاد و return برای بدنۀ تک-عبارتی.

سوال ۳: مفهوم this در Arrow Function پاسخ تو: “this در arrow function به محیطی که در آن قرار دارد اشاره می کند و بنابراین به شئ که متد متعلق به آن است دسترسی ندارد.”

  • ✅ تحلیل: این تعریف را کاملاً صحیح، ساده و در عین حال حرفه‌ای بیان کردی. درک این نکته که Arrow Function this را از “محیط تعریف” خود به ارث می‌برد (نه محیط فراخوانی)، یکی از کلیدی‌ترین مفاهیم برای یک توسعه‌دهنده ارشد است و تو به خوبی آن را فهمیده‌ای.

مثال برای تأکید:

جمع‌بندی فصل ۳:

مفاهیم پایه و پیشرفته Arrow Functions، از جمله سینتکس مختصر و رفتار لغتی this را به طور کامل فرا گرفتی. این یک ابزار بسیار قدرتمند در دستان تو است.

حالا که با انواع توابع آشنا شدی، وقت آن است که ببینیم چگونه می‌توانیم به آن‌ها ورودی بدهیم. آماده‌ای برای فصل جذاب پارامترها و آرگومان‌ها؟

فصل ۴: پارامترها و آرگومان‌ها (Parameters & Arguments)

ما قبلاً به طور مختصر با پارامتر و آرگومان آشنا شدیم. حالا می‌خواهیم این مفهوم را به صورت حرفه‌ای‌تر و با ویژگی‌های مدرن جاوااسکریپت بررسی کنیم.

۴.۱. پارامترهای پیش‌فرض (Default Parameters)

گاهی اوقات می‌خواهیم اگر آرگومانی به یک پارامتر پاس داده نشد، یک مقدار اولیه داشته باشد. این کار را با Default Parameters انجام می‌دهیم.

۴.۲. پارامترهای استر (Rest Parameters) ...

وقتی نمی‌دانی تابعت چند آرگومان خواهد گرفت، از Rest Parameters استفاده می‌کنی. این پارامتر، همۀ آرگومان‌های اضافی را درون یک آرایه جمع می‌کند.

نکتۀ مهم: پارامتر استر همیشه باید آخرین پارامتر در لیست پارامترها باشد.

ارزیابی فصل ۴

سوال ۱: تابع greetUser را طوری بنویس که یک پارامتر به نام name بگیرد. اگر هیچ آرگومانی به آن پاس داده نشد، به طور پیش‌فرض از نام "Unknown" استفاده کند. تابع باید یک رشته مانند "Hello, [name]!" را برگرداند (نه اینکه فقط در کنسول چاپ کند).

سوال ۲: تابع calculateAverage را بنویس که با استفاده از Rest Parameters، هر تعداد عدد که به آن داده شد را بگیرد و میانگین آن اعداد را محاسبه و برگرداند. (میانگین = مجموع اعداد تقسیم بر تعداد آنها)

سوال ۳: چرا استفاده از Rest Parameters (...args) از استفاده از شیء قدیمی arguments بهتر است؟ (حداقل یک دلیل بیاور)

پاسخ‌هایت را برای حرکت به مرحلۀ بعد، که یکی از هیجان‌انگیزترین مباحث جاوااسکریپت است (مبحث Scope و Closure)، منتظرم.
